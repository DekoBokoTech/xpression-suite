これは何？ XPressionのスクリプト（.NET / Visual Basic）で使えるオブジェクトモデル（API）の完全リファレンスです。全94クラス・3231メンバーを署名つきで収録しています。

ページの使い方：上の検索欄でクラス名・メンバー名・署名を横断検索できます。クラス行をクリックすると展開してメンバー一覧が出ます。バッジ → Property プロパティ（値の取得/設定）／Function 戻り値のある関数／Sub 戻り値のない手続き／Event イベント。署名は Visual Basic 表記です。

全体像：xpEngine（エンジン）→ Project → xpScene（シーン）→ オブジェクト（xpTextObject 等）→ xpMaterial（素材）という階層。多くの操作は「取得 → 設定」の形です。

最初の一歩（テキストを書き換える）：

dim obj as xpBaseObject
dim txt as xpTextObject
Scene.GetObjectByName "Text1", obj
txt = obj
txt.Text = "Hello"

コードを自動生成したいとき：右上の「Scriptビルダーを開く」から、アクションを選んで値を入れるだけでコードが作れます。

2つの文脈：シーン内の Script Editor（シーン内で Scene やオブジェクトを操作）と、外部制御用の .NET ライブラリ（外部アプリから Engine を操作）。VLの“裏側のコード”もこのAPIです。

What is this? A complete reference of the object model (API) available to XPression scripting (.NET / Visual Basic). It covers all 94 classes and 3231 members with signatures.

Using this page: search across class names, member names and signatures above. Click a class row to expand its members. Badges → Property get/set a value / Function returns a value / Sub no return value / Event. Signatures are Visual Basic.

Big picture: xpEngine → Project → xpScene → objects (xpTextObject, …) → xpMaterial. Most work is a "get → set" pattern.

First step (change a text):

dim obj as xpBaseObject
dim txt as xpTextObject
Scene.GetObjectByName "Text1", obj
txt = obj
txt.Text = "Hello"

Generate code automatically: use "Open Script Builder" (top right) — pick an action, fill in values, and it writes the code.

Two contexts: the in-scene Script Editor and the external .NET library. The code behind Visual Logic uses this same API.
